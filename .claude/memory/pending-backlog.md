项目在 `doc/08-待办建议-Backlog.md` 维护了一份待处理建议清单。

**工作机制：**
- 开发过程中 Claude 提出的建议，如果用户选择暂时不做，Claude 会写入此文档
- 下次用户打开新会话问"有什么之前建议我做但是还没做的"，Claude 读取此文档回答
- 用户做了就从清单移除，放弃的标注 `[已关闭]`

**为什么：** 避免好建议被遗忘，跨会话追踪待办改进项。

**如何应用：** 新会话开始时，如果用户问起待办建议，直接读 `doc/08-待办建议-Backlog.md` 的待处理列表。
