项目在 `doc/09-会话检查点.md` 维护会话检查点，每次会话结束时自动更新。

**工作机制：**
- 新会话开始 → 先读检查点知道"上次做到哪了、下一步做什么"
- 开发完成一个最小任务 → 更新检查点的"任务完成情况"
- 做了关键决策 → 记入"决策记录"
- 会话中断/结束 → 更新全部状态

**为什么：** 没有检查点的话，做到一半中断再开新窗口就是从头再来，之前推理的上下文全丢。

**如何应用：** 每次开始开发前读检查点，每次结束开发前写检查点。
