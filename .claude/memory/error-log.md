项目在 `doc/07-运行报错日志.md` 维护了一份运行报错日志，记录所有编译/运行/测试报错及其根因分析和解决方案。

**使用方式：**
- 遇到报错时，先在日志中搜索是否已有同类记录（按关键词或报错信息片段）
- 修复后，将新的报错按格式追加到该文档
- 高频错误会在文档底部汇总缩略版

**为什么：** 记录运行报错能避免重复踩坑，节省排查时间，特别对于跨 agent 协作的开发模式，能让不同角色共享经验。

**如何应用：** 每次修复一个报错后，整理根因和预防措施写入 `doc/07-运行报错日志.md`，并在涉及代码处添加注释标注预防措施。
