## 二、关键技术决策摘要

（从 `02-细节问题.md` 归纳）

| 决策点 | 结论 |
|--------|------|
| 追踪范围 | 所有前台 exe，无白名单 |
| 空闲阈值 | 15 分钟缓冲，恢复后延续上一条记录 |
| PS 文件路径 | 进程工作目录 + 文件句柄推断 |
| 终端/Claude Code 路径 | 窗口标题解析 + 父进程继承 |
| URL 精度 | 先做标题解析域名，后续扩展 |
| 日报格式 | Markdown，含时间线/项目/App/域名分布 |
| 内存目标 | < 50MB |
| 崩溃恢复 | 5 分钟窗口自动补录 |
| 隐私模式 | 正常记录 |
| 开机自启 | 注册表 `HKCU\...\Run` |

## 三、项目架构快照

```
Windows 11 桌面
    │
    ▼
ActivityMonitor.TrayApp (WPF)
  ├── TrayIcon          ← 系统托盘
  ├── Dashboard         ← 主面板：时间线 + 实时统计
  ├── ReportEditor      ← 日报编辑/导出
  ├── History           ← 历史浏览
  └── Settings          ← 设置
    │
    ▼
ActivityMonitor.Core
  ├── Tracking/         ← WindowTracker, IdleDetector, SleepDetector, CrashRecovery, ActivityEngine
  ├── Classification/   ← BrowserTracker, FileTracker, ProjectDetector, ActivityCategorizer, KeywordExtractor, TodayStatsService
  ├── Models/           ← ActivityEvent, DailySummary, WeeklySummary, MonthlySummary
  ├── Interfaces/       ← IActivityTracker, IActivityRepository, IActivityCategorizer 等
  └── Win32/            ← NativeMethods (P/Invoke)
    │
    ▼
ActivityMonitor.Data
  ├── SqliteContext     ← 建表/迁移
  └── Repositories/     ← CRUD + 聚合查询
```

## 四、6 个 Agent 角色定义

所有角色定义在 `.claude/agents/` 下，每个 agent 有严格的职责边界和工具限制：

| Agent | 文件名 | 负责模块 | 限制 |
|-------|--------|---------|------|
| 技术架构师 | `architect.md` | Wave 0: M1-M4 | 只做数据模型/接口/DB/Win32 |
| 后端引擎工程师 | `backend-engineer.md` | T1-T5 追踪引擎 | 不做 UI/分类/报表 |
| 内容分析工程师 | `content-analyst.md` | C1-C6 内容解析 | 不做引擎/UI/DB Schema |
| WPF 前端工程师 | `frontend-engineer.md` | U1-U5 界面 | 不做后端逻辑，可 Mock |
| 数据报表工程师 | `report-engineer.md` | R1-R5 报表 | 不做引擎/UI/分类 |
| 测试工程师 | `test-engineer.md` | E1-E5 测试 | 只写测试，不改生产代码 |
