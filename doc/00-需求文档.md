## 一、产品概述

一款 Windows 11 后台桌面活动追踪工具，自动记录用户在电脑前的一切前台操作，最终**一键导出 Markdown 格式的工作日报**。

### 核心理念

- **只看前台** — 只记录当前屏幕上的活动窗口，后台进程不纳入统计
- **不设白名单** — 所有 exe 一视同仁，分类是后处理逻辑
- **本地存储** — 所有数据保存在本地，不联网
- **单实例运行** — 只允许启动一个进程实例，禁止多开

---

## 二、用户角色

单人使用，当前 Windows 用户。

## 四、非功能需求

### N1 性能

> **引入版本：** v1.0（N1.1~N1.5）| **v1.1 新增** N1.6（1s 刷新降频策略）

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| N1.1 | 后台运行时**内存占用 < 50MB** | 任务管理器确认 |
| N1.2 | 用户无感知，不造成系统卡顿 | 轮询线程 `BelowNormal` 优先级 |
| N1.3 | 数据库批量写入：每 30 秒或缓存 50 条落盘一次 | 避免每条记录单次 IO |
| N1.4 | 空闲/睡眠状态下降低轮询频率到 10 秒一次 | 节省 CPU |
| N1.5 | 启动后延迟 10 秒开始追踪 | 让系统开机阶段优先完成其他加载 |
| N1.6 | Dashboard 1 秒刷新频率下不导致 UI 卡顿；若单次刷新计算耗时超过 **200ms**，自动降频至每 **2 秒**一次；空闲/睡眠场景下刷新频率统一降至每 **5-10 秒**一次（与 N1.4 协调联动）；降频后若连续 30 秒无卡顿（单次刷新 < 50ms），自动恢复 1 秒频率 | 滚动/操作流畅 |

| N3.3 | 睡眠/休眠自动暂停，唤醒自动恢复 |

## 五、功能边界（明确不做）

| 边界 | 说明 |
|------|------|
| 多显示器 | 当前只追踪单屏前台窗口 |
| 多用户 | 只追踪当前登录的 Windows 用户 |
| 多设备合并 | 暂不支持，如有需要手动在日报中补充 |
| 浏览器完整 URL | Phase 1 只从标题解析域名，浏览器扩展延后 |
| 网络同步 | 所有数据仅存本地，不支持云同步 |
| 屏幕截图 | 不录制屏幕内容 |
| 键盘记录 | 不记录按键内容（只记录窗口标题和 URL） |
| 文件系统监听 | 不做文件系统级别的事件监听（ReadDirectoryChangesW），通过窗口标题变化推断文件操作 |

## 六、用户交互流程

```
开机 → 自动启动（单实例） → 系统托盘（后台记录）
      ↓
今日首次打开 Dashboard → 查看实时统计 + 时间线（1 秒自动刷新）
      ↓
编辑记录（删除/合并/改分类/补充线下活动/项目重命名）
      ↓
预览日报 → 导出 Markdown（可选附带操作详情日志） → 手动补充总结
      ↓
点击关闭 × → 弹窗选择：加载到后台 / 彻底退出
      ↓
关闭窗口 → 托盘继续后台记录
      ↓
右键托盘 → 暂停监控 / 恢复监控 / 导出日报 / 退出
```

## 七、术语表

| 术语 | 含义 |
|------|------|
| activity_events | 核心事件表，每条记录代表一个连续的前台活动 |
| work_tag | 工作/非工作标记：`work` / `break` / `personal` / `unknown` |
| is_continued | 是否延续上一条记录（空闲后恢复场景） |
| project | 从窗口标题或进程路径推断出的项目文件夹名 |
| idle_threshold | 空闲阈值，默认 15 分钟 |
| crash_recovery | 崩溃 5 分钟内重启后的自动补录机制 |
| user_tags | 用户自定义类别标签（F6.5），与系统内置类别互补 |
| TimeSegment | 半小时时间段聚合单元（F8.7 / F6.6），用于类别继承和时段统计 |
| lock_screen | 锁屏事件（F4.6），自动归类为 break |
