# 活动监控器 (Activity Monitor) — 需求文档

> 创建日期：2026-07-20
> 版本：v1.0
> 关联文档：`01-架构设计.md`（实现方案）、`02-细节问题.md`（需求问答记录）

---

## 一、产品概述

一款 Windows 11 后台桌面活动追踪工具，自动记录用户在电脑前的一切前台操作，最终**一键导出 Markdown 格式的工作日报**。

### 核心理念

- **只看前台** — 只记录当前屏幕上的活动窗口，后台进程不纳入统计
- **不设白名单** — 所有 exe 一视同仁，分类是后处理逻辑
- **本地存储** — 所有数据保存在本地，不联网

---

## 二、用户角色

单人使用，当前 Windows 用户。

## 三、功能需求

### F1 前台窗口追踪

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F1.1 | 以 2 秒为间隔轮询当前前台窗口，记录窗口标题、进程名、进程路径、进程 ID | P0 |
| F1.2 | **所有 exe 无差别追踪**，不设白名单或黑名单 | P0 |
| F1.3 | 后台运行的无窗口进程（系统服务、托盘常驻等）不纳入统计 | P0 |
| F1.4 | 窗口切换时自动结束上一条记录、创建新记录；窗口不变时累加计时 | P0 |
| F1.5 | 每天从 00:00 开始统计，跨天重置 | P0 |

### F2 浏览器活动追踪

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F2.1 | 从浏览器窗口标题解析当前标签页的页面标题和域名 | P0 |
| F2.2 | 支持 Chrome、Edge、Firefox 的标题格式解析 | P0 |
| F2.3 | 从窗口标题中提取关键词（去停用词） | P1 |
| F2.4 | 无痕/隐私窗口同样正常记录（无痕也是工作的一部分） | P1 |
| F2.5 | 【未来增强】通过浏览器扩展获取完整 URL 路径 | P3 |

### F3 文件与应用活动追踪

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F3.1 | 从窗口标题推断正在操作的文件名 | P0 |
| F3.2 | **推断文件所属的项目文件夹**：通过窗口标题 + 进程工作目录 + 文件句柄综合判断 | P0 |
| F3.3 | 支持常见应用：VS Code、Photoshop、Office、终端/CMD、PowerShell、远程桌面 | P0 |
| F3.4 | **PS 文件路径获取**：进程工作目录 + 打开文件句柄推断完整文件路径 | P1 |
| F3.5 | **终端/Claude Code 起始目录**：通过终端窗口标题解析当前路径，子进程继承父进程工作目录 | P1 |
| F3.6 | 所有 Python 生成的 exe、未知 exe 均正常记录窗口标题和进程名 | P0 |
| F3.7 | 项目检测通用兜底：取进程路径所在文件夹名或最近的 `.git` 上级目录作为项目名 | P1 |

### F4 空闲与中断处理

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F4.1 | 通过 `GetLastInputInfo` 检测用户最后一次键盘/鼠标输入 | P0 |
| F4.2 | 空闲阈值设为 **15 分钟**（用户思考缓冲期） | P0 |
| F4.3 | 空闲期间暂停计时，恢复后**延续上一条活动记录**而非创建新记录 | P0 |
| F4.4 | 空闲时段在日报中显示时长，用户可以编辑补充描述（如"午休"、"线下开会"） | P1 |
| F4.5 | 监听系统电源事件，睡眠/休眠时自动暂停，唤醒后自动恢复，标记为"睡眠" | P0 |

### F5 崩溃恢复

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F5.1 | 通过退出标记文件检测程序是否非正常退出 | P1 |
| F5.2 | 若崩溃后 **5 分钟内**重新启动，自动补录缺失记录；超过 5 分钟则从启动时刻开始记录 | P1 |
| F5.3 | 崩溃恢复的记录打上标记以便区分 | P1 |

### F6 活动分类

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F6.1 | 自动判断每条活动的类别：`web` / `file` / `app` / `idle` / `sleep` | P0 |
| F6.2 | 自动判断工作/非工作（`work_tag`）：基于域名 + 窗口标题关键词 + 用户历史标记 | P1 |
| F6.3 | 支持用户手动重新标记任意记录的类别和工作/非工作属性 | P1 |
| F6.4 | 支持**多层下钻统计**：`工作/非工作 → 项目/App/域名 → 明细时间线` | P1 |

### F7 实时统计面板

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F7.1 | 动态统计从 **00:00 到当前时间**的各项时间占比 | P1 |
| F7.2 | 按**软件**（exe）聚合显示时长和占比 | P1 |
| F7.3 | 按**项目**（文件夹/Git 仓库）聚合显示时长和占比 | P1 |
| F7.4 | 按**网站域名**聚合显示时长和占比 | P1 |
| F7.5 | 支持按工作/非工作切换视角 | P1 |
| F7.6 | Dashboard 定时刷新以保证数据实时性 | P1 |

### F8 Dashboard 窗口

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F8.1 | 当日时间线视图：按时间顺序展示所有活动记录 | P0 |
| F8.2 | 时间线支持**编辑、删除、合并**记录 | P1 |
| F8.3 | 支持**插入线下活动**记录（如开会、不在电脑前的工作） | P1 |

### F9 日报导出

| ID | 需求描述 | 优先级 |

## 四、非功能需求

### N1 性能

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| N1.1 | 后台运行时**内存占用 < 50MB** | 任务管理器确认 |
| N1.2 | 用户无感知，不造成系统卡顿 | 轮询线程 `BelowNormal` 优先级 |
| N1.3 | 数据库批量写入：每 30 秒或缓存 50 条落盘一次 | 避免每条记录单次 IO |
| N1.4 | 空闲/睡眠状态下降低轮询频率到 10 秒一次 | 节省 CPU |
| N1.5 | 启动后延迟 10 秒开始追踪 | 让系统开机阶段优先完成其他加载 |

### N2 隐私与安全

| ID | 需求描述 |
|----|----------|
| N2.1 | **所有数据本地存储**，绝不联网发送 |
| N2.2 | 不录制屏幕、不捕获键盘内容 |
| N2.3 | 数据库路径：`%LOCALAPPDATA%\ActivityMonitor\data.db` |
| N2.4 | 用户可随时暂停监控 |
| N2.5 | 用户可删除/修改任意记录 |

### N3 可靠性

| ID | 需求描述 |
|----|----------|
| N3.1 | 开机自启，默默在后台运行 |
| N3.2 | 崩溃后 5 分钟内重启可自动补录缺失数据 |
| N3.3 | 睡眠/休眠自动暂停，唤醒自动恢复 |

## 五、功能边界（明确不做）

| 边界 | 说明 |
|------|------|
| 多显示器 | 当前只追踪单屏前台窗口 |
| 多用户 | 只追踪当前登录的 Windows 用户 |
| 多设备合并 | 暂不支持，如有需要手动在日报中补充 |
| 浏览器完整 URL | Phase 1 只从标题解析域名，浏览器扩展延后 |
| 网络同步 | 所有数据仅存本地，不支持云同步 |
| 屏幕截图 | 不录制屏幕内容 |
| 键盘记录 | 不记录按键内容（只记录窗口标题和 URL） |

## 六、用户交互流程

```
开机 → 自动启动 → 系统托盘（后台记录）
      ↓
今日首次打开 Dashboard → 查看实时统计 + 时间线
      ↓
编辑记录（删除/合并/改分类/补充线下活动）
      ↓
预览日报 → 导出 Markdown → 手动补充总结
      ↓
关闭窗口 → 托盘继续后台记录
```

## 七、术语表

| 术语 | 含义 |
|------|------|
| activity_events | 核心事件表，每条记录代表一个连续的前台活动 |
| work_tag | 工作/非工作标记：`work` / `break` / `personal` / `unknown` |
| is_continued | 是否延续上一条记录（空闲后恢复场景） |
| project | 从窗口标题或进程路径推断出的项目文件夹名 |
| idle_threshold | 空闲阈值，默认 15 分钟 |
| crash_recovery | 崩溃 5 分钟内重启后的自动补录机制 |
