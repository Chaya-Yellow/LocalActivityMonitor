# 需求细节问题清单

> 用于补全"一键导出当天工作日报"这个目标下隐藏的细节。
> 全部已回答（2026-07-20），已依据答案更新 `00-架构设计.md`。

---

## 一、范围与边界

### Q1: 什么才算"工作活动"？

你会在同一台电脑上做**非工作**的事情吗（刷视频、看新闻、购物）？日报里是否要：
- A) 全部记录，你自己手动区分
- ~~B) 通过域名/应用白名单自动归类~~ ← **最终方案（结合A**）
- C) 只记录你标记为"工作"的应用/网站，其他的忽略

**A1：**
B 方案为主，但需要补充：像 bilibili.com 这类网站有时是工作、有时不是，所以**网页标题也要一同记录**，方便后续辨别哪些是工作哪些是非工作。

**→ 架构影响**：新增 ActivityCategorizer 模块，根据域名/进程名 + 窗口标题 智能分类，并支持用户重新标记。

---

### Q2: 你一天通常使用哪些应用？

Adobe 系列、IDE、办公、通讯、终端、远程工具（RDP/Todesk/向日葵）、模型工具（CC Switch）、科学上网（星云/v2rayN）、输入法、图片查看器……以及**他人用 Python 生成的 exe**。

**A2：**
不需要穷举，直接**追踪所有前台窗口的 exe**，后台进程不纳入统计。关键原则：**只看当前屏幕上的窗口是什么，不看后台在运行什么。**

**→ 架构影响**：WindowTracker 改为通用追踪器，移除白名单概念。

---

### Q3: 是否有多显示器？

**A3：** 目前只有一个显示器，无需多屏合并逻辑。

## 二、日报内容与格式

### Q4: 日报期望包含哪些信息？

- A) 时间线视图
- B) 分类统计
- **C) 详细到具体文件/网页** ← 选择
- D) 关键词云

**A4：**
选 C。**编辑文件时需要告知是在哪个项目文件夹下操作。**

**→ 架构影响**：FileTracker 需要能从窗口标题+进程路径推断出所属的项目文件夹。

---

### Q5: 导出格式？

**A5：** Markdown 格式。

---

### Q6: 导出后要支持手动编辑吗？

**A6：**
支持。**并且需要能补充不在电脑前的操作的记录**，例如线下开会。

**→ 架构影响**：需要 ReportEditor UI，支持增删改活动记录、插入线下活动。

## 三、空闲与中断

### Q7: 空闲时间怎么处理？

**A7：**
选 B（显示空闲时长），并且**可以编辑补充这段时间做了什么事情**。

**→ 架构影响**：空闲记录支持用户编辑描述。

### Q8: 如果开了一个网页/文件后一直没动（人离开了但窗口没切换）？

**A8：**
文件没动可能是在**思考**，思考时间定为 **15 分钟**内。15 分钟后仍然没有操作就暂停，**回来后算上一个活动的延续**。

**→ 架构影响**：空闲阈值从 5 分钟改为 15 分钟；恢复后延续上一条记录而非新建。

### Q10: 是否允许手动修改/删除已记录的活动？

**A10：** 允许，删记录、合并记录、改类别/描述都支持。

## 八、异常场景

### Q19: 电脑休眠/睡眠恢复后？

**A19：** 自动恢复监控，空闲时间置为"睡眠"。

**→ 架构影响**：新增 SleepDetector，监听系统电源事件。

### Q20: 程序崩溃后重启？

**A20：**
如果崩溃关闭后在 **5 分钟内**重新启动该程序，**自动补录**崩溃期间的缺失记录。

**→ 架构影响**：新增 CrashRecoveryService，启动时检测上次非正常退出时间窗口。

## 九、需求补充

### Q21：想要实现的功能

A21：动态统计从00:00-当前时间使用不同软件、不同类别的时间占比（后续可能还会有更新）

**→ 架构影响**：新增 TodayStatsService 按需查询当天 activity_events 实时聚合（按进程名、按类别）；Dashboard 需新增实时统计面板（软件分布 + 类别分布 + 进度环/百分比）。

### Q22：PS 文件 / CMD 启动路径的追踪

PS（Photoshop）打开一个 psd 文件时，窗口标题通常只显示文件名而不显示文件夹路径。通过 CMD 进入 Claude Code 时，也需要知道是从哪个文件夹进入的。

你希望怎么处理？
- A) 只记录 PS 文件名，文件夹由你手动标注
- B) 通过进程的**工作目录**（Working Directory）或**打开的文件句柄**来推断文件夹路径
- C) 通过窗口标题分析，有路径就显示路径，没有就只显示文件名

**A22：**
选 B，尽量自动推断 PS 文件的文件夹路径。CMD/终端启动的程序（如 Claude Code），通过终端窗口标题或进程工作目录获取起始文件夹。

**→ 架构影响**：FileTracker 增强——PS 通过进程工作目录 + 打开的文件句柄获取文件完整路径；终端工具通过窗口标题解析当前目录（`cmd.exe - C:\path`）；子进程（如 Claude Code）继承父进程的目录。

---

### Q23：工作/非工作分类的精细度

在日报和实时统计中，你希望看到哪些层级的时间占比？
- A) 按**软件**（exe）占比
- B) 按**项目**（文件夹 / Git 仓库）占比
- C) 按**网站域名**占比
- D) 以上全部，并且**工作/非工作**分开统计

**A23：**
以上全部。工作/非工作分类下都需要能细化到：
- 每个项目占用了多少时间
- 每个 App 占用了多少时间
- 每个网站占用了多少时间

**→ 架构影响**：实时统计和日报都需要多层下钻：`工作/非工作 → 项目/App/域名 → 明细时间线`。TodayStatsService 需要按 project、process_name、domain 分别聚合。

---

### Q24：运行性能与内存占用

监控程序需要在后台长期运行，你怎么看待它对系统资源的影响？
- A) 无所谓，现在的电脑性能过剩
- B) 尽量轻量，内存占用控制在 50MB 以内，不能感觉到卡顿
- C) 极端轻量，像任务管理器一样不占用资源

**A24：**
选 **B**。尽量减小内存使用，防止卡顿。毕竟是长期运行的。

**→ 架构影响**：架构和实现层面需要：2s 定时器而非 while(true) 高频循环、批量写入 SQLite（每 30s 或 N 条一次）、轮询线程优先级设为 BelowNormal、避免反射和 LINQ 高分配操作、数据库查询走索引。
