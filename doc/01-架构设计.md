## 一、项目概述

在 Windows 11 后台运行的桌面活动追踪工具。**追踪所有出现在前台窗口的应用程序**（不限于浏览器和编辑器），记录用户在每个窗口/文件/网页上停留了多久、操作了什么，最终**一键导出 Markdown 格式的工作日报**，并支持手动编辑补充。

核心设计原则：
- **只看前台** — 只记录当前屏幕上的活动窗口，后台进程不纳入统计
- **不拘泥白名单** — 所有 exe 一视同仁，分类是后处理逻辑
- **所有数据本地存储**，绝不联网

## 二、技术选型

| 层面 | 选择 | 理由 |
|------|------|------|
| 语言/框架 | **C# .NET 8 WPF** | 原生 Windows API 调用、系统托盘、ETW 支持 |
| UI | **WPF + System Tray** | 托盘图标（运行/暂停/空闲状态），悬停显示今日时长 |
| 数据库 | **SQLite (via Microsoft.Data.Sqlite)** | 零配置、单文件、本地存储 |
| 后台运行 | **BackgroundService** | 随托盘进程启动/停止 |
| 日志/聚合 | **SQLite 聚合 + 定时任务** | 日/周/月三级聚合 |

## 四、核心追踪模块

### 4.1 WindowTracker（通用前台窗口追踪器）

- **定位**：不做白名单过滤，**所有出现在前台的 exe 全部记录**
- **轮询间隔**：每 **2 秒**一次
- **工作流**：
  1. `GetForegroundWindow()` → 前台窗口句柄
  2. `GetWindowThreadProcessId()` → 进程 ID + exe 路径
  3. `GetWindowText()` → 窗口标题
  4. 与当前活动记录对比：
     - 窗口相同 → 累加活跃时长
     - 窗口变了 → 结束上条记录，创建新记录（类别自动识别）
- **排除规则**：后台无窗口进程、系统托盘弹窗等不纳入

### 4.2 BrowserTracker（浏览器标签页追踪器）

- **当前策略（Phase 1-2）**：从窗口标题解析域名
  - Chrome: `"Linux 内核编译参数 - Google Chrome"` → `linux 内核编译参数`
  - Edge: `"GitHub - Microsoft Edge"` → `github.com`
  - Firefox: `"百度搜索xxx - Mozilla Firefox"`
- **未来增强（Phase 3+）**：通过浏览器扩展获取完整 URL
- **关键词提取**：从标题去除停用词，提取有意义词组

### 4.3 FileTracker（文件与项目追踪器）

追踪**所有前台窗口**，编辑器、PS、IDE、终端、远程桌面、Python exe……全部记录。

从**窗口标题 + 进程工作目录 + 打开的文件句柄**三个维度推断文件路径和项目：

| 应用 | 窗口标题示例 | 路径获取方式 |
|------|-------------|-------------|
| **VS Code** | `app.py - project-name - Visual Studio Code` | 标题解析 → 项目名 `project-name` |
| **Photoshop** | `设计稿_v3.psd @ 100% (RGB/8)` | 标题仅提供文件名；通过进程**工作目录** + **打开的文件句柄**获取完整路径 |
| **终端 / CMD** | `cmd.exe - C:\project\code\` 或 `管理员: C:\Windows\System32\cmd.exe` | 标题解析当前目录路径 |
| **PowerShell** | `Windows PowerShell - C:\project\code` | 标题解析当前目录路径 |
| **Claude Code（终端内）** | 运行在终端中，继承终端的当前工作目录 | 从终端窗口标题获取起始目录，子进程追踪父进程的工作目录 |
| **远程桌面** | `远程桌面连接 - 192.168.1.100` | 记录远程目标地址 |
| **Python exe / 未知 exe** | 任意窗口标题 | 记录窗口标题 + 进程路径 + 工作目录，后续可自定义标注 |
| **Windows 资源管理器** | `项目文件` 或 `项目文件 > 子文件夹` | 标题即文件夹名 |

**项目检测（ProjectDetector）**：
- **VS Code**：标题 `文件名 - 项目名 - Visual Studio Code` → 提取项目名
- **终端类**：标题 `cmd.exe - C:\Users\name\project\` → 提取最后一级文件夹作为项目名
- **PS/编辑器**：通过进程的工作目录推断 → 取路径中最近的 `.git` 上级目录或最后两级文件夹
- **Claude Code**：追踪其父进程（CMD/PowerShell/VS Code 终端）的工作目录
- **通用兜底**：从进程路径取所在文件夹名作为项目名

> **注**：PS 文件完整路径获取有难度（窗口标题只显示文件名）。Phase 1 先用进程工作目录推断文件夹，若精度不够 Phase 2 再引入 `NtQueryInformationProcess` + 句柄查询。

### 4.4 IdleDetector（空闲检测器）

- `GetLastInputInfo()` 获取上次用户输入时间
- **空闲阈值：15 分钟**（用户可能在思考，给予缓冲）
- 空闲期间暂停计时
- **恢复后延续上一条活动记录**（不计为新记录）
- 空闲时段在日报中显示时长，**支持用户编辑补充描述**（如"午休"、"线下开会"）

### 4.5 SleepDetector（睡眠/休眠检测）

- 监听系统电源事件 `WM_POWERBROADCAST`
- 睡眠/休眠时自动暂停监控
- 恢复时自动继续，空闲时间段标记为**"睡眠"**
- 无需用户手动操作

### 4.6 CrashRecoveryService（崩溃恢复）

- 程序启动时检测上次退出是否为**非正常退出**（通过退出标记文件）
- 若检测到非正常退出，且距上次记录结束时间 < **5 分钟**：
  - 自动补录这 5 分钟的缺失记录
  - 标记为"崩溃恢复"
- 若距上次记录结束时间 > 5 分钟：从启动时刻开始正常记录

### 4.7 ActivityCategorizer（活动分类器）

根据以下规则自动判断活动类别：

| 规则 | 判断依据 | 类别 |
|------|----------|------|
| 进程名在浏览器列表 | `chrome.exe`, `msedge.exe`, `firefox.exe` | `web` |
| 窗口标题含"无痕"/"InPrivate" | 标题检测 | `web` + 隐私标记 |
| 进程名为已知编辑器 | `code.exe`, `ps.exe`, `notepad++.exe`, `winword.exe` 等 | `file` |
| 进程名为远程工具 | `mstsc.exe`, `todesk.exe`, `sunlogin.exe` | `app` → 标记远程 |
| 系统锁屏/睡眠 | 电源事件 | `sleep` |
| 其他所有前台 exe | 默认 | `app` |
| 超过空闲阈值 | GetLastInputInfo | `idle` |

**工作/非工作分类**：基于域名 + 窗口标题关键词 + 用户历史标记综合判断。
- 用户可以在 Dashboard 中重新标记某条记录为"工作"或"非工作"
- 系统学习用户习惯，后续自动分类
- 分类后支持**多层下钻统计**：`工作/非工作 → 项目/App/域名 → 明细时间线`

### 4.8 性能优化指南

后台监控长期运行，目标内存占用 < **50MB**，用户无感知。

| 优化维度 | 措施 |
|----------|------|
| **轮询频率** | 2s 间隔，使用 `System.Timers.Timer` 而非 while(true) 循环 |
| **线程优先级** | 轮询线程设为 `ThreadPriority.BelowNormal` |
| **数据库写入** | 批量写入，每 **30 秒** 或缓存满 **50 条** 落盘一次，避免每条记录单次 INSERT |
| **数据库查询** | activity_events 建索引 `(start_time, process_name, category)`；日报查询只扫当天数据 |
| **内存分配** | 避免反射、避免 LINQ 高分配操作（如频繁 `.ToList()`、`.OrderBy()`）；使用 `StringBuilder` |
| **窗口查询** | `GetForegroundWindow()` 本身极轻量（微秒级），仅在窗口变化时做全量解析 |
| **空闲/睡眠检测** | 检测到空闲后**降低轮询频率**到 10s/次，节省 CPU |
| **启动时** | 延迟 10s 启动追踪引擎，让系统开机阶段先完成其他加载

## 五、数据模型

```sql
-- 核心事件表
CREATE TABLE activity_events (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    start_time      TEXT NOT NULL,             -- ISO 8601
    end_time        TEXT,                       -- NULL = 进行中
    duration_ms     INTEGER DEFAULT 0,         -- 实际活跃毫秒数

    -- 活动类别
    category        TEXT NOT NULL,             -- 'web' | 'file' | 'app' | 'idle' | 'sleep'
    work_tag        TEXT DEFAULT 'unknown',    -- 'work' | 'break' | 'personal' | 'unknown'
    sub_category    TEXT,                       -- 细化类别: 'editor' | 'terminal' | 'remote' | 'browser'

    -- 窗口信息
    window_title    TEXT,
    process_name    TEXT,                       -- exe 文件名
    process_path    TEXT,                       -- exe 完整路径
    process_id      INTEGER,

    -- 内容详情
    detail          TEXT,                       -- URL / 文件路径 / 详细描述
    domain          TEXT,                       -- 浏览器域名
    project         TEXT,                       -- 所属项目/仓库名
    keywords        TEXT,                       -- JSON 数组

    -- 元数据
    is_continued    INTEGER DEFAULT 0,          -- 是否为延续上一次的记录（空闲后恢复）
    is_private      INTEGER DEFAULT 0,          -- 隐私模式标记
    is_crash_recovered INTEGER DEFAULT 0,       -- 崩溃恢复标记

    -- 用户编辑
    edited_title    TEXT,                       -- 用户修改后的标题
    edited_desc     TEXT,                       -- 用户补充的描述（如空闲时段说明）
    user_category   TEXT                        -- 用户重新标记的类别
);

-- 每日聚合表（加速 Dashboard）
CREATE TABLE daily_summaries (
    date                TEXT PRIMARY KEY,        -- 'YYYY-MM-DD'
    total_active_ms     INTEGER DEFAULT 0,
    total_idle_ms       INTEGER DEFAULT 0,
    total_sleep_ms      INTEGER DEFAULT 0,
    app_breakdown       TEXT,                    -- JSON { "code.exe": 3600000, ... }
    domain_breakdown    TEXT,                    -- JSON { "github.com": 1800000, ... }
    project_breakdown   TEXT,                    -- JSON { "project-name": 7200000, ... }
    work_ms             INTEGER DEFAULT 0,       -- 标记为"工作"的总时长
    break_ms            INTEGER DEFAULT 0,       -- 标记为"休息"的总时长
    keyword_cloud       TEXT,                    -- JSON 词频
    raw_report          TEXT,                    -- 预生成的日报 Markdown 草稿
    user_notes          TEXT                     -- 用户补充的日报内容
);

-- 每周聚合表
CREATE TABLE weekly_summaries (
    week_start          TEXT PRIMARY KEY,        -- 'YYYY-MM-DD' (周一)
    week_end            TEXT,
    total_active_ms     INTEGER DEFAULT 0,
    total_idle_ms       INTEGER DEFAULT 0,
    app_breakdown       TEXT,                    -- JSON
    domain_breakdown    TEXT,                    -- JSON
    project_breakdown   TEXT,                    -- JSON
    avg_daily_hours     REAL DEFAULT 0
);

-- 每月聚合表
CREATE TABLE monthly_summaries (
    month               TEXT PRIMARY KEY,        -- 'YYYY-MM'
    total_active_ms     INTEGER DEFAULT 0,
    total_idle_ms       INTEGER DEFAULT 0,
    app_breakdown       TEXT,                    -- JSON
    domain_breakdown    TEXT,                    -- JSON
    project_breakdown   TEXT,                    -- JSON
    avg_daily_hours     REAL DEFAULT 0
);

-- 设置表
CREATE TABLE settings (
    key   TEXT PRIMARY KEY,
    value TEXT
);
-- 预置: idle_threshold_minutes=15, retention_days=30, auto_start=true
```

## 七、关键技术难点与解决方案

| 难点 | 解决方案 |
|------|----------|
| 获取任意 exe 的前台窗口 | `GetForegroundWindow()` + `GetWindowThreadProcessId()`，无白名单 |
| 浏览器 URL（域名级） | 窗口标题截断解析（`"xxx - Google Chrome"`） |
| 浏览器 URL（完整 URL） | Phase 3+ 通过浏览器扩展/ CDP 协议获取 |
| 编辑器/PS 项目文件夹推断 | VS Code 标题提取项目名；终端路径解析；PS 通过进程工作目录 + 文件句柄；Git 仓库检测 |
| 终端 / Claude Code 起始目录 | 解析终端窗口标题 `cmd.exe - C:\path`；子进程继承父进程工作目录 |
| PS 文件完整路径获取 | 窗口标题只显示文件名；通过 `GetProcessWorkingDirectory` + 打开文件句柄查询补全路径 |
| 所有 exe 的类型识别 | 基于进程名 + 窗口标题规则的分类器，用户可重标 |
| 非工作 vs 工作区分 | 域名 + 标题关键词 + 用户历史标记的权重分类；支持工作/非工作下按项目/App/域名下钻 |
| 空闲判断 | `GetLastInputInfo` + 15 分钟阈值 |
| 空闲后恢复 | 延续上一条记录（`is_continued=1`），不新建 |
| 睡眠/休眠 | 监听 `WM_POWERBROADCAST` + `SetThreadExecutionState` |
| 崩溃恢复 | 退出标记文件 + 5 分钟窗口自动补录 |
| 日报编辑 | WPF DataGrid 直接修改记录，插入线下活动行 |
| 周/月聚合 | 定时任务（每日凌晨）跑 SQLite 聚合 |
| 数据保留策略 | 配置 retention_days，定时任务清理旧数据 |
| 当日实时统计（多级下钻） | TodayStatsService：按 `work_tag → project/process_name/domain` 三级聚合，Dashboard 定时刷新 |
| 内存控制 < 50MB | 2s 定时器而非高频循环；批量写入（30s/50条）；`BelowNormal` 线程优先级；空闲时降低轮询频率到 10s |
| 开机自启 | 注册表 `HKCU\...\Run` |

## 十一、隐私与安全

- **所有数据本地存储**，绝不联网发送
- 不录制屏幕、不捕获键盘内容
- 无痕浏览器窗口正常记录（你确认了这是工作需求）
- 数据库路径：`%LOCALAPPDATA%\ActivityMonitor\data.db`
- 用户可随时暂停监控、删除/修改任何记录
- 用户可自定义数据保留时长
